Setup & Invite Guide

Integrate Count Check directly into your Discord server. Follow the quick guide below to invite the bot and configure its permissions to activate slash commands.

Step 1: Invite the Bot

Click your customized OAuth2 generator link to authorize the bot. The bot requires the following standard, read-only gateway permissions to operate:

  • Read Messages / View Channels: Enables coordinate command routing.
  • Send Messages & Embed Links: Required to dispatch formatted portfolio ledgers.
  • Attach Files: Critical. Required to upload Nearest-Neighbor high-resolution graphics and charts.
  • Use External Emojis: Required to render custom caskets, skills, and item icons.
🔗 Click Here to Invite the Bot

Step 2: Grant Channel Permissions & Role Hierarchy

To make sure the bot has adequate gateway scope to process, compile, and send analytics in your channels, execute these settings adjustments:

  1. Grant Channel Access: Ensure that the bot's integration role (Count Check) has standard permissions enabled in any channels you want users to trigger commands in. This includes View Channel, Send Messages, and Attach Files.
  2. Set Role Hierarchy: Open Server Settings ➔ Roles. Drag the bot's integration role (Count Check) so that it is positioned higher in the list than any standard server roles you want it to interact with or manage. If the bot is below a role, it cannot manage members of that role.

Step 3: Force Sync & Use Slash Commands

To register and display Count Check's autocomplete command menus instantly inside your server, perform these quick synchronization actions:

  1. Trigger Global Sync: Execute the prefix command !globalsync (or !sync) in any text channel. This forces the Discord gateway API to flush and rebuild the command definitions.
  2. Client Reload: If autocomplete commands do not immediately appear in your slash (/) menu, reload your Discord client (Ctrl + R on desktop) to refresh the cached command schema. You can now use any of the public analytical, sizing, and simulation commands.
